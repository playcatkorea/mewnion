# 이용약관 및 개인정보처리방침 링크 수정 완료

## 🔧 수정 내용

회원가입 및 로그인 페이지에서 이용약관과 개인정보처리방침 링크가 제대로 작동하지 않던 문제를 수정했습니다.

### 문제점
1. **잘못된 경로 연결**
   - 이용약관: `/notice` (잘못됨) → `/terms` (올바름)
   - 개인정보처리방침: `/news` (잘못됨) → `/privacy` (올바름)

2. **Form 내부 Button 충돌**
   - form 안에 있는 `<button type="button">`이 form submit과 충돌 가능성
   - onClick 이벤트가 제대로 작동하지 않을 수 있음

### 해결 방법
- `<button>` → `<a>` 태그로 변경
- 새 탭에서 열리도록 설정 (`target="_blank"`)
- 보안 속성 추가 (`rel="noopener noreferrer"`)
- window.open() 사용으로 확실한 새 탭 열기

## 📝 수정된 파일

### 1. 회원가입 페이지
**파일**: `src/pages/signup/page.tsx`

**변경 전**:
```tsx
<button
  className="underline"
  style={{color: '#ffff00'}}
  onClick={() => navigateTo('/notice')}
  type="button"
>
  이용 약관
</button>
```

**변경 후**:
```tsx
<a
  href="/terms"
  target="_blank"
  rel="noopener noreferrer"
  className="underline hover:opacity-80"
  style={{color: '#ffff00'}}
  onClick={(e) => {
    e.preventDefault();
    window.open('/terms', '_blank', 'noopener,noreferrer');
  }}
>
  이용 약관
</a>
```

```tsx
  onClick={() => navigateTo('/notice')}
>
  이용약관

```

**변경 후**:

## ✨ 개선 사항

### 1. 사용자 경험 향상
- 약관을 새 탭에서 열어 회원가입/로그인 진행 중인 내용이 유지됨
- 약관을 읽고 돌아올 때 입력한 정보가 사라지지 않음

### 2. 보안 강화
- `rel="noopener noreferrer"` 속성으로 보안 취약점 방지
- window.opener 접근 차단
- referrer 정보 보호

### 3. 접근성 개선
- `<a>` 태그 사용으로 시맨틱 HTML 준수
- 스크린 리더 사용자에게 더 명확한 정보 제공
- hover 효과 추가 (`hover:opacity-80`)

## 🎯 작동 원리

### 링크 클릭 시 동작 순서
1. 사용자가 링크 클릭
2. `onClick` 핸들러 실행
3. `e.preventDefault()`로 기본 동작 방지
4. `window.open('/terms', '_blank', 'noopener,noreferrer')` 실행
5. 새 탭에서 약관 페이지 열림
6. 원래 탭은 그대로 유지

### 장점
- **Form Submit 방지**: `e.preventDefault()`로 form 제출 차단
- **새 탭 열기**: 사용자가 약관을 읽고 돌아올 수 있음
- **Fallback 제공**: JavaScript 비활성화 시에도 `href` 속성으로 작동
- **보안**: `noopener,noreferrer`로 보안 강화

## ⚠️ 주의사항

1. **팝업 차단 확인**
   - 일부 브라우저는 팝업 차단 설정이 활성화되어 있을 수 있음
   - 사용자에게 팝업 허용 안내 필요

2. **모바일 환경**
   - 모바일에서는 새 탭이 아닌 새 창으로 열릴 수 있음
   - 뒤로가기 버튼으로 원래 화면으로 돌아갈 수 있음

3. **JavaScript 비활성화**
   - JavaScript가 비활성화된 경우 `href` 속성으로 같은 탭에서 열림
   - 이 경우 뒤로가기로 돌아와야 함
